Introduction


A typical toad.js user interface divides the code running in the browser into three horizontal layers to ease testing and modification.

The dependencies between these layers go only in one direction: Interface → Application → Domain.

Let's jump right into an example:

The Soda Machine
Flavour
Quantity ml
Fill Bottle

The Interface Layer

The Interface Layer defines what the user sees and interacts with using CSS/HTML.

Here is the simplified HTML for "The Soda Machine" above: 

                   The Soda Machine

Flavour  <tx-select model="flavour">
             <option value="CLASSIC">Classic</option>
             <option value="CHERRY">Cherry</option>
             <option value="VANILLA">Vanilla</option>
         </tx-select>

Quantity <tx-text model="quantity"></tx-text> ml

<tx-button action="fill">Fill Bottle</tx-button>
  • The tx-* HTML Tags are Web Components provided by toad.js and are used instead of the standard HTML Tags.
  • The model and action attributes reference values and functions in the next layer, the Application Layer.
  • It is also possible to use JSX instead of HTML so that the IDE can provide feedback that the referenced values and functions exits and have a type suitable to be viewed and edited by the HTML Tag.
  • The View Layer can be maintained by a UX designer without the help of a coder as the tx-* HTML tags do not need any code. (Actually, they don't even have event attributes like onmousedown or onkeydown.)
  • These HTML Tags are also commonly known as Widgets, Views, Windows or User Interface Components.

The Application Layer

The Application Layer controls how the application behaves. 
const flavour = new EnumModel<Flavour>(Flavour)
flavour.value = Flavour.CLASSIC
bind("flavour", flavour)

const quantity = new NumberModel(
    defaultSize,
    {min: 0, max: 1500})
bind("quantity", quantity)

action("fill", () => {
    fillBottle(quantity.value, flavour.value/1000.0))
    quantity.value = defaultSize
})

  • A Model is a thing which represents another thing and is used instead of it.

    The models in the Application Layer are commonly known as

    • View Models, meaning they are a model of a View in the layer above (View Layer) or,
    • Value Models, meaning they are a model of a value in the layer below (Domain Layer).
  • The classes EnumModel, NumberModel and Action are models provided by toad.js and can also be disabled/enabled, validate input, provide error messages, etc.
  • The Application Layer does not need CSS/HTML and can be tested outside the browser so that tests do not have to deal with the complexity of the browser's handling of HTML, events, etc.

The Domain Layer

The Domain Layer contains all the code and data regarding what the application is actually about (minus the user interface). 

enum Flavour {
    CLASSIC, CHERRY, VANILLA
}
const defaultSize = 0.33
function fillBottle(flavour: Flavour, litre: number) {
    fetch(`http://127.0.0.1/fill/${flavour}/${litre}`)
}
  • The code and data in this Layer are also known as Domain Model.

Action

Actions are models without values (aside from properties like enabled, label, etc.)

An Action's modfied signal will be triggered on a user interaction, e.g. a push button being pressed.

TypeScript

                
      class MyNode implements TreeNode {
        label: string
        next?: MyNode
        down?: MyNode
        static counter = 0
        constructor() {
            this.label = `#${MyNode.counter++}`
        }
      }
      
      class MyTreeAdapter extends TreeAdapter<MyNode> {
          override getDisplayCell(col: number, row: number) {
              return this.model && this.treeCell(row, this.model.rows[row].node.label)
          }
      }
    
      TreeAdapter.register(MyTreeAdapter, TreeNodeModel, MyNode)
      let model = new TreeNodeModel(MyNode)
      model.addSiblingAfter(0)
      model.addChildAfter(0)
      model.addChildAfter(1)
      model.addSiblingAfter(2)
      model.addSiblingAfter(1)
      model.addChildAfter(4)
      model.addSiblingAfter(0)
      bind("tree", model)
