Introduction

toad.js aims to separate the frontend implementation inside the browser into three horizontal layers.

Layer Description
View Layer

Defines the visible structure and look of the application. Usually written in HTML or similar.

Create this layer in direct collaboration between User, Coder and UX.

Draft it with pen and paper, then write it down in code and use Hot Module Replacement and Fast Refresh to see what it looks in the browser.

Drafting the UI in Photoshop, Abstract, Figma or similar is an Agile Anti-Pattern.

Do not define behaviour in this layer, avoid using JavaScript. Instead use HTML elements (Views) which reference a model (ViewModel) in the the Application Layer.

E.g. 

  Flavour: <tx-popupmenu model="flavour"></tx-popupmenu> 
  Bottle Size: <tx-text model="volume"></tx-text> liters
  <tx-button action="fillBottle">Fill bottle</tx-button>

You should be able to write HTML which does not need to be covered by tests. The HTML elements on the other hand should be covered by tests running in a variety of browsers. Since they are highly reusable and need to adhere to the selected Design System, their creation and maintance maybe done by a separate team.

The views included in toad.js are implemented as WebComponents taking inspiration from Adobe's Spectrum Design System/Style Guide and use IBM's Plex font family.

Presentation Layer
(aka. ViewModel, Application Model, Controller, ...)

Defines the behaviour of the application's user interface.

This layer can tested outside a browser as there are no Views in this layer.

See also the article on the Presentation Model by Martin Fowler.

Domain Layer
(aka. Domain Model, Model, ...)

Defines the data and behaviour of the domain.

E.g. the flavours available in the Soda machine, minimum to maximum bottle size and a function to start dispensing the soda into the bottle.

This layer can tested outside a browser.

Action Model

Actions are models without values (aside from properties like enabled, label, etc.)

An Action's modfied signal will be triggered on a user interaction, e.g. a push button being pressed.

      class Book {
        title: string = ""
        author: string = ""
        year: number = 1970
      }
  
      const bookList = [
          { title: "The Moon Is A Harsh Mistress", author: "Robert A. Heinlein", year: 1966 },
          { title: "Stranger In A Strange Land", author: "Robert A. Heinlein", year: 1961 },
          { title: "The Fountains of Paradise", author: "Arthur C. Clarke", year: 1979 },
          { title: "Rendezvous with Rama", author: "Arthur C. Clarke", year: 1973 },
          { title: "2001: A Space Odyssey", author: "Arthur C. Clarke", year: 1968 },
          { title: "Do Androids Dream of Electric Sheep?", author: "Philip K. Dick", year: 1968 },
          { title: "A Scanner Darkly", author: "Philip K. Dick", year: 1977 },
          { title: "Second Variety", author: "Philip K. Dick", year: 1953 },
      ]
      
      class BookAdapter extends ArrayAdapter<ArrayModel<Book>> {
          override getColumnHeads() { return ["Title", "Author", "Year"] }
          override getRow(book: Book) { return refs(book, "title", "author", "year") }
      }
      
      TableAdapter.register(BookAdapter, ArrayModel, Book)
      const model = new ArrayModel<Book>(bookList, Book)
      bind("books", model)

      class MyNode implements TreeNode {
        label: string
        next?: MyNode
        down?: MyNode
        static counter = 0
        constructor() {
            this.label = `#${MyNode.counter++}`
        }
      }
      
      class MyTreeAdapter extends TreeAdapter<MyNode> {
          override getDisplayCell(col: number, row: number) {
              return this.model && this.treeCell(row, this.model.rows[row].node.label)
          }
      }
    
      TreeAdapter.register(MyTreeAdapter, TreeNodeModel, MyNode)
      let model = new TreeNodeModel(MyNode)
      model.addSiblingAfter(0)
      model.addChildAfter(0)
      model.addChildAfter(1)
      model.addSiblingAfter(2)
      model.addSiblingAfter(1)
      model.addChildAfter(4)
      model.addSiblingAfter(0)
      bind("tree", model)
